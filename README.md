# LED Canvas

## 简介

`extension.led_canvas` 是一个用于 Skydimo 的扩展页面工具。

它的核心目标是把多个真实设备、输出或分段放到同一张 LED 画布中，再将整张画布注册成一个虚拟设备，方便统一预览、统一布光和统一套用灯效。

你可以把它理解成一个“灯光布局编辑器 + 虚拟设备控制面板”。

## 主要功能

- 多布局管理：支持创建、切换、重命名、删除多个布局。
- 设备画布编排：可以把设备输出或分段添加到画布中，自由摆放位置和尺寸。
- 旋转与镜像：已放入画布的设备支持旋转；选中设备后还可以做水平镜像，调整灯珠方向。
- 单设备亮度控制：每个 placement 都可以单独设置亮度。
- 虚拟画布设备注册：可将当前布局注册为一个虚拟矩阵设备。
- 虚拟设备灯效控制：注册后可直接控制虚拟设备的开关、灯效和灯效参数。
- 状态回读：会同步虚拟画布输出当前的开关、灯效和参数状态。
- 原始 JSON 调试：布局管理底部可展开查看虚拟画布输出的原始 JSON，便于排查字段结构。
- 冲突检测：当多个布局或 placement 占用同一输出的同一批 LED 时，会做占用处理并显示可用/被阻塞状态。
- 撤销重做：支持通过快捷键撤销和重做画布编辑操作。

## 适用场景

- 想把多个设备拼成一整块逻辑画布来做统一灯效。
- 想把键盘、灯带、风扇、桌面灯板等设备映射到同一个二维布局里。
- 想把虚拟设备暴露给 Core，再像普通设备一样控制它的灯效。
- 需要调试虚拟设备输出层的 `mode` / `power` / `effective_params` 等字段。

## 基本使用方法

### 1. 选择或创建布局

打开 `LED Canvas` 页面后：

- 在顶部选择已有布局，或新建一个布局。
- 不同布局互相独立，适合保存不同的桌面/设备摆放方案。

### 2. 把设备加入画布

左侧设备树会列出当前可用设备及其输出/分段：

- 点击 `+` 可以把输出或分段加入画布。
- 如果一个输出下有多个分段，可以只添加某个分段。
- 已加入画布的项会显示为可选中状态。

### 3. 编辑设备位置与方向

在中间画布中可以直接编辑已加入的设备：

- 拖动：移动位置。
- 缩放：调整尺寸。
- 旋转：通过旋转手柄调整角度。
- 镜像：在左侧设备树中选中某个已加入设备后，会在亮度区域下方显示“镜像”按钮，点击可水平镜像灯珠方向。
- 亮度：选中后可在设备树内调整该设备的单独亮度。

如果开启网格吸附，移动和缩放时会更容易对齐。

### 4. 注册虚拟画布设备

右侧 `Layout Manager / 布局管理` 面板里可以注册当前布局的虚拟设备：

- 打开“注册虚拟设备”开关。
- 注册后，当前布局会对应一个虚拟设备端口，输出为 `canvas`。
- 之后这张画布就可以像普通设备输出一样接收灯效和电源状态。

### 5. 控制虚拟设备灯效

注册完成后，可以在布局管理中：

- 开关虚拟设备电源。
- 选择灯效。
- 调整灯效参数。
- 重置当前灯效参数。

这些设置会作用在虚拟设备的 `canvas` 输出上，而不是仅作用于设备根节点。

### 6. 查看回读与调试信息

如果想确认 Core 当前实际返回了什么数据：

- 打开布局管理底部的“原始输出 JSON”。
- 这里会显示虚拟画布 `canvas` 输出的原始 payload。
- 当灯效、参数或开关状态看起来不对时，可以先看这里的字段。

## 典型工作流

1. 新建一个布局。
2. 从左侧把多个设备输出或分段加入画布。
3. 调整它们的位置、大小、旋转、镜像和亮度。
4. 开启网格吸附，整理整体布局。
5. 在右侧注册虚拟画布设备。
6. 给虚拟设备选择灯效并调参。
7. 如有异常，展开底部原始输出 JSON 做调试。

## 快捷操作

- `Ctrl/Cmd + Z`：撤销
- `Ctrl/Cmd + Shift + Z` 或 `Ctrl/Cmd + Y`：重做

## 数据与同步说明

- 画布上的 placement 改动会自动同步到后端配置。
- 当前活动布局切换时，前端会从后端重新 hydrate 画布状态。
- 虚拟设备状态会从 `canvas` 输出回读，而不是只读设备根节点。
- 单个 placement 的亮度、方向和矩阵映射会随布局一起保存。

## 目录结构

```text
extension.led_canvas/
├─ init.lua             # 后端逻辑：布局、虚拟设备注册、路由、回读
├─ manifest.json        # 插件元信息
├─ page/                # 前端页面
│  ├─ src/
│  │  ├─ components/    # 设备树、画布、布局管理等 UI
│  │  ├─ lib/           # bridge、canvas store、i18n
│  │  └─ types/         # 前端类型定义
│  └─ dist/             # 构建产物
├─ lib/
│  ├─ routing.lua       # 画布到真实 LED 的路由计算
│  ├─ json.lua
│  └─ persist.lua
├─ locales/             # 插件元信息语言包
└─ data/                # 运行时数据目录
```

## 补充说明

- 如果设备结构发生变化，某些已添加到画布的 placement 可能会被标记为 stale，需要移除后重新添加。
- 镜像本质上会改写该 placement 的 LED 矩阵映射，因此会同时影响画布预览和实际输出顺序。
- 如果虚拟设备已注册，但灯效字段看起来异常，优先检查“原始输出 JSON”里 `canvas` 输出下的 `mode` 和 `power`。
